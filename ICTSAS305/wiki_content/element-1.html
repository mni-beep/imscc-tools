Element 1: Review Client Support Issues

üîç What This Element Covers

Learn how to effectively investigate, document, and resolve client support issues.

Performance Criteria

  1. Check for new problems logged by client
  2. Check previous logs for similar problems
  3. Investigate and document support issues
  4. Notify client of results and provide advice
  5. Obtain client feedback and make changes

üìù Key Steps

1. Check New Problems

Review the ticketing system or support log for new issues. Look for:

  • Problem description
  • Priority level
  • Client contact details

2. Review Previous Logs

Search historical records to see if similar issues have occurred before. This helps you:

  • Find proven solutions
  • Identify recurring problems
  • Save time on investigation

3. Investigate & Document

Thoroughly investigate the issue and document your findings clearly. Include:

  • Steps taken to investigate
  • Root cause analysis
  • Proposed solution
  • Any limitations or risks

4. Communicate Results

Use clear, jargon-free language when explaining technical issues to clients. Focus on:

  • What the problem is (in simple terms)
  • Why it happened
  • How you'll fix it
  • Expected timeframe
üí° Pro Tip: Always follow up with clients after implementing a solution. This ensures the issue is fully resolved and builds trust.
